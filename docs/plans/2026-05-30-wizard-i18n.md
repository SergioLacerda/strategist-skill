# Wizard i18n — 3 Language Dimensions Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace the single `Language` field in `WizardConfig` with four explicit language fields (UI, docs, chat, code) and make the install wizard prompt in the user's chosen language.

**Architecture:** `WizardConfig` gains `UILanguage`, `DocLanguage`, `ChatLanguage`, `CodeLanguage`. The wizard asks UI language first (bilingual prompt), then all subsequent prompts use string bundles keyed on that choice. `active.yaml` gains a nested `language:` block with all four values.

**Tech Stack:** Go 1.22+, `bufio.Reader` stdin injection (existing pattern), `testify/assert`

**Step 2: Add `promptValidatedRaw`**

The existing `promptValidated` hardcodes the error format string. Add a raw variant that accepts it as a parameter (so bundles can supply it), then rewrite `promptValidated` to call it:

```go
// promptValidatedRaw prompts and retries until a valid value is entered.
// errFmt is a format string accepting (value, accepted) — e.g. "  Invalid value %q. Accepted: %s\n".
func promptValidatedRaw(r *bufio.Reader, question, defaultVal string, accepted []string, errFmt string) (string, error) {
	for {
		val, err := prompt(r, question, defaultVal)
		if err != nil {
			return "", err
		}
		for _, a := range accepted {
			if strings.EqualFold(val, a) {
				return strings.ToLower(val), nil
			}
		}
		fmt.Printf(errFmt, val, strings.Join(accepted, ", "))
	}
}

func promptValidated(r *bufio.Reader, question, defaultVal string, accepted []string) (string, error) {
	return promptValidatedRaw(r, question, defaultVal, accepted, "  Invalid value %q. Accepted: %s\n")
}
```

Note: `promptValidatedRaw` returns `strings.ToLower(val)`. For language values like `pt-BR` this would corrupt the case. The callers in `runWizard` normalise the case explicitly after calling `promptValidatedRaw`, so `strings.ToLower` is fine in the helper (it lowercases `pt-br` which the caller then re-normalises to `pt-BR`).
