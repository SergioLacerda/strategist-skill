## Task 2 — Create `compile-knowledge-index.sh`

**Description:** Shell script that reads `knowledge.index.yaml` and writes `.compiled/.index.gz` (gzipped JSON inverted tag index).

**Files affected:**
- `.strategist/scripts/compile-knowledge-index.sh` (new)

**Implementation:** Full script spec in `design.md §3.2`.

Key requirements:
- Accept `$1` = knowledge_index_yaml path, `$2` = output.gz path
- Use `yq -o json` to convert YAML, `jq` to build inverted index, `gzip` to compress
- Embed `sources` block with source file path → mtime
- Cross-platform mtime: try `stat -c %Y` (Linux), fall back to `stat -f %m` (macOS)
- Exit non-zero on missing input; log to stderr
- Idempotent: safe to re-run

**Completion criterion:** Running the script on the actual `knowledge.index.yaml` produces a valid `.gz` file; `gunzip -c .compiled/.index.gz | jq '.tags'` returns a non-empty object; `gunzip -c .compiled/.index.gz | jq '.sources'` contains the source file path and mtime.

## Task 5 — Create `check-stale.sh`

**Description:** Shell script that checks whether a compiled artifact is fresh relative to its recorded source files. Used by the agent (via Bash tool) before each fast-path read.

**Files affected:**
- `.strategist/scripts/check-stale.sh` (new)

**Implementation:** Full script spec in `design.md §3.1`.

Key requirements:
- Accept `$1` = path to compiled `.gz` artifact
- Exit 0 if fresh, 1 if stale or absent
- Check for `.manifest.gz` presence (absent → treat all as stale)
- Read `sources` block from the artifact itself using `gunzip -c | jq`
- For each source: check file exists and `actual_mtime <= recorded_mtime`
- Cross-platform mtime: same `stat` fallback as compile scripts
- No side effects (read-only)

**Completion criterion:**
- `check-stale.sh .compiled/.config.gz` returns 0 immediately after a fresh compile
- After `touch active.yaml`, returns 1
- After `rm .compiled/.manifest.gz`, returns 1
- With a non-existent artifact path, returns 1
